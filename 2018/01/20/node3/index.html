类

以前把数据和操作数据处理的方法打包是写一个对象，用 key(操作数据的函数名称): value(函数的实现) 的形式
现在用类来处理

  • static 静态方法
  • 类名.方法名()的形式调用, this 指向类， this.name 指向类名(字符串)
  • 实例方法里面 this.constructor 代表这个类

  • 继承 class User extends Model (Model 的方法和属性 User 都可以使用)

    • 继承的时候要先调用 super 的方法, 才可以使用 this

    • 创建实例方法
      用 create 代替 new 创建实例, 好处是可以都定制这个实例, 比如创建实例的同时，把这个实例保存起来

ORM Object Relational Mapping

对象关系映射（英语：(Object Relational Mapping，简称ORM），是一种程序技术，用于实现面向对象编程语言里不同类型系统的数据之间的转换。数据库的对象和JSON结合起来。

重写 toString

原因: object.toString() 返回的是 [object Object]
当我们发送 html 的时候, 需要发送字符串的格式 所以要把数据里面的对象转化为字符串
用 String.replace() 的时候， 会自动先调用 toString() 方法，如果替换的是对象, 就会变成上面的格式

重写就是用 JSON 把对象改成字符串形式, 给类添加一个 toString(), 覆盖掉默认的 toString()
这样使用 replace 的时候, 对象就可以变成一个可以显示的字符串了
