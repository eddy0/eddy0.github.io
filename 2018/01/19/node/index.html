http 组成

  • protocol: http, https
  • host:
  • port: 80(http), 443(https)
  • path: /

dns 服务器 —- domain name server

http 协议: 超文本传输协议

  1. 服务器按规定的格式发送给文本数据(请求)到服务器
  2. 服务器解析请求, 按照规定的格式返回文本数据(响应到浏览器)到浏览器
  3. 浏览器解析得到的数据， 并做相应的处理

格式:

CLIENT

  1. 建立连接
  2. 监听连接
  3. 接受数据
  4. 处理数据
  5. 发送数据
  6. 关闭连接

SERVER

  1. 配置相关参数
  2. 创建服务器
  3. 监听连接
  4. 接受数据
  5. 处理数据
  6. 发送数据
  7. 关闭连接

data 返回的是一个 Buffer 类型， 用来处理二进制数据 toString()
Buffer 默认是大概有 1024 个数据左右

server.destroy() 如果没有的话， 浏览器会一直在loading 页面等待服务器发完数据
另外一种解决方法就是在header里面用 content-length 来显示多少数据 如果小于发送的数据长度， 内容显示不全; 如果大于发送的数据的长度， 浏览器会一直loading 一般计算方法${body.length} 或者 ${body.byteLength} 如果是 buffer 类型
