前后端分离

这里只的是开发环境上的分离

  1. 普通开发模式
    前端写客户端, 后端写 api 服务端

  • 问题:

    • 前端需要搭建后端的开发环境, 然后运行后端的项目
    • 前后端对接的时候后端改动一个地方, 前端可能需要更新项目代码, git pull 之后发现需要更新更多的代码

  • 另外一种方式: 前后端在两个环境里面, 后端设置好 cors 解决跨域的问题

  1. 前后端分离

  • 办法:

    • 用 node 做转发服务器,
    • 通过配置 nginx 的方式在实现分离

  • 流程:

    • node 把请求发送到 api server, api server 处理好请求并且返回响应
    • node 接受 api server 放回的响应, 并且把这个响应发送到浏览器端
    • 浏览器接受响应, 渲染页面
